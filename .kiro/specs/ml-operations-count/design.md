# Design Document

## Overview

Данный документ описывает проектное решение для удаления неиспользуемого поля "ФактическаяДлительностьЧас" из модели Операция_МЛ в системе WebAPI_OTK. Изменения затрагивают модель данных, DTO классы, контроллеры API и схему базы данных.

## Architecture

Система использует следующую архитектуру:
- **Backend**: ASP.NET Core Web API
- **ORM**: Entity Framework Core
- **Database**: SQLite
- **Pattern**: Repository pattern через DbContext
- **API Style**: RESTful API с контроллерами

Изменения будут внесены на следующих уровнях:
1. **Data Layer**: Модель Операция_МЛ (Models/Операция_МЛ.cs)
2. **DTO Layer**: DTO классы для операций (DTOs/ОперацияМЛDtos.cs)
3. **API Layer**: Контроллер OperationRouteListController
4. **Database Layer**: Миграция Entity Framework для удаления колонки

## Error Handling

### Возможные ошибки при миграции

1. **Ошибка применения миграции**
   - Причина: База данных заблокирована или недоступна
   - Решение: Система использует `EnsureDeleted()` и `EnsureCreated()` в Program.cs, что пересоздает БД

2. **Потеря данных**
   - Причина: Удаление колонки приводит к потере данных в этом поле
   - Решение: Данные в поле ФактическаяДлительностьЧас будут удалены (это ожидаемое поведение)

3. **Ошибки компиляции**
   - Причина: Возможные ссылки на удаленное свойство в других частях кода
   - Решение: Проверить все файлы проекта на наличие ссылок на ФактическаяДлительностьЧас

## Implementation Notes

### Порядок выполнения изменений

1. **Обновить модель Операция_МЛ**
   - Удалить свойство ФактическаяДлительностьЧас из Models/Операция_МЛ.cs

2. **Обновить конфигурацию Entity Framework**
   - Удалить конфигурацию precision для ФактическаяДлительностьЧас из Models/Model1.cs

3. **Обновить DTO классы**
   - Удалить свойство ФактическаяДлительностьЧас из всех DTO в DTOs/ОперацияМЛDtos.cs

4. **Обновить контроллер**
   - Удалить присваивание ФактическаяДлительностьЧас в OperationRouteListController.cs

5. **Обновить frontend**
   - Удалить столбец "Длительность (ч)" из таблицы операций в wwwroot/js/pages/ml-closure.js

6. **Создать миграцию**
   - Выполнить команду `dotnet ef migrations add RemoveDurationFromOperationML`

7. **Проверить миграцию**
   - Убедиться, что миграция содержит только удаление колонки ФактическаяДлительностьЧас

8. **Применить миграцию**
   - Миграция будет применена автоматически при запуске приложения через `EnsureCreated()`

### Обратная совместимость

Данное изменение **не является обратно совместимым**:
- Клиенты API, которые отправляют поле ФактическаяДлительностьЧас, не получат ошибку (поле будет проигнорировано)
- Клиенты API, которые ожидают поле ФактическаяДлительностьЧас в ответах, больше не получат это поле

### Влияние на существующие данные

- Все данные в колонке ФактическаяДлительностьЧас будут удалены из базы данных
- Это необратимая операция (если не создавать резервную копию БД)
- Учитывая, что в Program.cs используется `EnsureDeleted()`, база данных пересоздается при каждом запуске, потеря данных не критична

## Risks and Mitigation

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Потеря важных данных | Низкая | Высокое | БД пересоздается при каждом запуске, данные тестовые |
| Ошибки в клиентском коде | Средняя | Среднее | Обновить frontend код для удаления столбца |
| Ошибки миграции | Низкая | Среднее | Использование EnsureDeleted/EnsureCreated |
| Несовместимость API | Средняя | Низкое | JSON сериализация игнорирует неизвестные поля |
| Отображение пустого столбца | Низкая | Низкое | Удалить столбец из frontend таблицы |
