# Requirements Document

## Introduction

Данная спецификация описывает изменения в модели данных операций маршрутных листов (Операция_МЛ) в системе WebAPI_OTK. Изменения направлены на упрощение структуры данных путем удаления неиспользуемого поля длительности.

## Glossary

- **Операция_МЛ**: Отдельная операция в рамках маршрутного листа
- **Система**: WebAPI_OTK - веб-приложение для управления маршрутными листами
- **База данных**: SQLite база данных, используемая для хранения данных системы
- **Entity Framework**: ORM-фреймворк для работы с базой данных
- **Миграция**: Изменение схемы базы данных с помощью Entity Framework

## Requirements

### Requirement 1

**User Story:** Как разработчик системы, я хочу удалить неиспользуемое поле "ФактическаяДлительностьЧас" из модели Операция_МЛ, чтобы упростить структуру данных

#### Acceptance Criteria

1. THE Система SHALL удалить свойство "ФактическаяДлительностьЧас" из модели Операция_МЛ
2. THE Система SHALL удалить колонку "ФактическаяДлительностьЧас" из таблицы базы данных
3. THE Система SHALL применить миграцию базы данных для удаления поля
4. THE Система SHALL обновить все DTO классы, связанные с Операция_МЛ, для удаления поля длительности
5. THE Система SHALL обновить контроллеры API для корректной работы без поля длительности

### Requirement 2

**User Story:** Как администратор системы, я хочу, чтобы изменения схемы базы данных применялись автоматически при запуске приложения, чтобы не выполнять миграции вручную

#### Acceptance Criteria

1. THE Система SHALL создать новую миграцию Entity Framework для изменений схемы
2. WHEN приложение запускается, THE Система SHALL автоматически применить все миграции к базе данных
3. THE Система SHALL сохранить существующие данные в таблице Операция_МЛ при применении миграций
4. IF миграция не может быть применена, THEN THE Система SHALL записать ошибку в лог
