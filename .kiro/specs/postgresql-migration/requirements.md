# Миграция на PostgreSQL - Требования

## Обзор
Миграция системы ОТК с SQL Server на PostgreSQL для устранения проблем с подключением и обеспечения кроссплатформенности.

## Технические требования

### Структура данных
- Все таблицы должны использовать PostgreSQL типы данных
- Использовать SERIAL для автоинкрементных ID
- Использовать VARCHAR вместо NVARCHAR
- Использовать TIMESTAMP для дат с временем
- Использовать DATE для дат без времени
- Использовать DECIMAL для денежных значений

### Тестовые данные
- Данные должны быть реалистичными
- Даты должны быть актуальными (2025-2026)
- Связи между таблицами должны быть корректными
- Должны быть примеры всех статусов и состояний

### Безопасность
- Не использовать пароли в открытом виде в примерах
- Использовать переменные окружения для чувствительных данных
- Рекомендовать создание отдельного пользователя БД для приложения

## Зависимости
- PostgreSQL 12+ (рекомендуется 15+)
- Npgsql.EntityFrameworkCore.PostgreSQL 8.0+
- .NET 8.0

## Риски и ограничения
- Необходимо обновить миграции EF Core для PostgreSQL
- Возможны различия в SQL синтаксисе между SQL Server и PostgreSQL
- Требуется установка PostgreSQL на машине разработчика
