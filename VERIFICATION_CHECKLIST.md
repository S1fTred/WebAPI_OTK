# Контрольный список проверки базы данных

## Шаг 1: Запуск сервера

1. Откройте проект в Rider IDE
2. Запустите сервер (кнопка Run или F5)
3. Дождитесь сообщения в консоли:
   - "Добавлено X операций к МЛ" (от AddOperationsToML)
   - "Обновлено X операций с новыми датами" (от UpdateOperationDates)
4. Остановите сервер

## Шаг 2: Проверка файлов базы данных

Запустите в PowerShell:
```powershell
./check_database_simple.ps1
```

Должны быть созданы файлы:
- ✓ `WebAPI_OTK/otk.db` - основной файл базы данных
- ✓ `WebAPI_OTK/otk.db-shm` (опционально) - shared memory
- ✓ `WebAPI_OTK/otk.db-wal` (опционально) - write-ahead log

## Шаг 3: Проверка структуры таблицы Операция_МЛ

### Вариант A: Через код (рекомендуется)

Запустите сервер и откройте в браузере:
```
http://localhost:5000/api/OperationRouteList?млId=1
```

Проверьте JSON-ответ на наличие полей:
```json
{
  "датаВыдачи": "2026-02-14T08:00:00",
  "датаИсполнения": "2026-02-15T11:30:00",
  "датаЗакрытия": "2026-02-18T08:00:00"
}
```

### Вариант B: Через DB Browser for SQLite

1. Скачайте и установите: https://sqlitebrowser.org/
2. Откройте файл `WebAPI_OTK/otk.db`
3. Перейдите на вкладку "Database Structure"
4. Найдите таблицу `Операция_МЛ`
5. Проверьте наличие колонок:
   - ✓ `ДатаВыдачи` (TEXT, nullable)
   - ✓ `ДатаИсполнения` (TEXT, nullable)
   - ✓ `ДатаЗакрытия` (TEXT, nullable)

## Шаг 4: Проверка данных

### Проверка количества записей

В DB Browser for SQLite выполните запросы:

```sql
-- Всего операций (должно быть ~40-50)
SELECT COUNT(*) FROM Операция_МЛ;

-- Операции с заполненными новыми датами (должно быть ~40-50)
SELECT COUNT(*) FROM Операция_МЛ WHERE ДатаВыдачи IS NOT NULL;

-- Всего МЛ (должно быть 30)
SELECT COUNT(*) FROM МЛ;

-- Всего сотрудников (должно быть 12)
SELECT COUNT(*) FROM Сотрудник;
```

### Проверка примеров данных

```sql
-- Посмотреть первые 5 операций с новыми датами
SELECT 
    ID,
    МЛID,
    ДатаНачала,
    ДатаВыдачи,
    ДатаИсполнения,
    ДатаЗакрытия,
    Статус
FROM Операция_МЛ
LIMIT 5;
```

Ожидаемый результат:
- ДатаВыдачи должна быть на 1 день раньше ДатаНачала
- ДатаИсполнения должна равняться ДатаОкончания (если операция завершена)
- ДатаЗакрытия должна быть на 3 дня позже ДатаНачала

## Шаг 5: Проверка интерфейса

1. Запустите сервер
2. Откройте в браузере: `http://localhost:5000/ml-closure.html`
3. Войдите в систему (любой логин/пароль)
4. Найдите любой МЛ и откройте его
5. В таблице операций должны быть колонки:
   - ✓ Дата Выдачи
   - ✓ Дата Исполнения
   - ✓ Дата Закрытия
6. Старые колонки "Дата начала" и "Дата окончания" должны отсутствовать

## Возможные проблемы

### Проблема: База данных не создается

**Решение:**
1. Проверьте права доступа к папке `WebAPI_OTK`
2. Проверьте, что файл `appsettings.json` содержит правильную строку подключения
3. Проверьте логи сервера на наличие ошибок

### Проблема: Новые поля отсутствуют

**Решение:**
1. Удалите файлы базы данных:
   ```powershell
   Remove-Item WebAPI_OTK/otk.db*
   ```
2. Перезапустите сервер

### Проблема: Даты не заполнены

**Решение:**
1. Проверьте логи сервера на сообщение "Обновлено X операций с новыми датами"
2. Если сообщения нет, проверьте файл `UpdateOperationDates.cs`
3. Удалите базу и пересоздайте её
