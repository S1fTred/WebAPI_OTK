# База данных PostgreSQL для системы ОТК

## Содержимое директории

### SQL скрипты
- **schema.sql** - Создание схемы БД (таблицы, индексы, представления, триггеры)
- **data.sql** - Тестовые данные для демонстрации системы

### Документация
- **SETUP_POSTGRESQL.md** - Подробная инструкция по настройке PostgreSQL
- **PROGRAM_CS_CHANGES.md** - Инструкция по изменению Program.cs
- **README.md** - Этот файл

## Быстрый старт

### 1. Создать базу данных
```bash
psql -U postgres -c "CREATE DATABASE otk_db;"
```

### 2. Выполнить схему
```bash
psql -U postgres -d otk_db -f schema.sql
```

### 3. Загрузить данные
```bash
psql -U postgres -d otk_db -f data.sql
```

## Структура базы данных

### Справочные таблицы
- **Должность** - Справочник должностей (7 записей)
- **Изделие** - Справочник изделий (4 записи)
- **ТипОперации** - Справочник типов операций (10 записей)
- **Оборудование** - Справочник оборудования (7 записей)

### Основные таблицы
- **Сотрудник** - Сотрудники предприятия (12 записей)
- **ДСЕ** - Детали и сборочные единицы (14 записей)
- **МЛ** - Маршрутные листы (8 записей)
- **Операция_МЛ** - Операции маршрутных листов (25+ записей)
- **ПремиальныеКоэффициенты** - Премиальные коэффициенты (9 записей)
- **Зарплата** - Расчет зарплаты (примеры)

## Особенности схемы

### Индексы
Созданы индексы для оптимизации частых запросов:
- По статусу МЛ (открыт/закрыт)
- По датам создания и выполнения
- По связям между таблицами
- По активности коэффициентов

### Представления (Views)
- **v_ActivePremKoef** - Активные премиальные коэффициенты на текущую дату
- **v_OpenML** - Открытые маршрутные листы с подробной информацией

### Триггеры
- **trg_calculate_duration** - Автоматический расчет фактической длительности операции при установке даты окончания

### Функции
- **calculate_operation_duration()** - Расчет длительности операции в часах

## Проверка данных

После выполнения скриптов проверьте:

```sql
-- Подключиться к БД
psql -U postgres -d otk_db

-- Проверить таблицы
\dt

-- Проверить количество записей
SELECT 'Должность' as table_name, COUNT(*) FROM "Должность"
UNION ALL
SELECT 'Изделие', COUNT(*) FROM "Изделие"
UNION ALL
SELECT 'Сотрудник', COUNT(*) FROM "Сотрудник"
UNION ALL
SELECT 'ДСЕ', COUNT(*) FROM "ДСЕ"
UNION ALL
SELECT 'МЛ', COUNT(*) FROM "МЛ"
UNION ALL
SELECT 'Операция_МЛ', COUNT(*) FROM "Операция_МЛ";

-- Проверить активные коэффициенты
SELECT * FROM "v_ActivePremKoef";

-- Проверить открытые МЛ
SELECT * FROM "v_OpenML";

-- Выйти
\q
```

## Резервное копирование

### Создать резервную копию
```bash
pg_dump -U postgres -d otk_db -f backup_$(date +%Y%m%d).sql
```

### Восстановить из резервной копии
```bash
psql -U postgres -d otk_db -f backup_20260306.sql
```

## Очистка данных

Если нужно очистить все данные и начать заново:

```sql
-- Подключиться
psql -U postgres -d otk_db

-- Очистить все таблицы
TRUNCATE TABLE "Зарплата" CASCADE;
TRUNCATE TABLE "Операция_МЛ" CASCADE;
TRUNCATE TABLE "МЛ" CASCADE;
TRUNCATE TABLE "ПремиальныеКоэффициенты" CASCADE;
TRUNCATE TABLE "ДСЕ" CASCADE;
TRUNCATE TABLE "Изделие" CASCADE;
TRUNCATE TABLE "Сотрудник" CASCADE;
TRUNCATE TABLE "Должность" CASCADE;
TRUNCATE TABLE "ТипОперации" CASCADE;
TRUNCATE TABLE "Оборудование" CASCADE;

-- Выйти
\q

-- Загрузить данные заново
psql -U postgres -d otk_db -f data.sql
```

## Удаление базы данных

Если нужно полностью удалить БД:

```bash
# Отключить все соединения
psql -U postgres -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname = 'otk_db';"

# Удалить БД
psql -U postgres -c "DROP DATABASE otk_db;"
```

## Миграция с SQL Server

Если вы мигрируете с SQL Server:
1. Экспортируйте данные из SQL Server
2. Выполните `schema.sql` для создания структуры
3. Импортируйте данные в PostgreSQL
4. Или используйте готовые тестовые данные из `data.sql`

## Производительность

### Рекомендации
- Регулярно выполняйте `VACUUM ANALYZE` для оптимизации
- Мониторьте размер БД: `SELECT pg_size_pretty(pg_database_size('otk_db'));`
- Проверяйте медленные запросы: `EXPLAIN ANALYZE SELECT ...`

### Оптимизация
```sql
-- Анализ таблиц
ANALYZE;

-- Очистка и анализ
VACUUM ANALYZE;

-- Пересоздание индексов (если нужно)
REINDEX DATABASE otk_db;
```
