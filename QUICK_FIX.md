# Быстрое исправление ошибки 500

## Что сделано

✅ Улучшена обработка ошибок - теперь страницы не падают при ошибках API
✅ Убраны множественные toast-уведомления
✅ Добавлены понятные сообщения вместо технических ошибок

## Что нужно сделать

### 1. Проверьте, что SQL Server запущен

```powershell
# Проверка статуса
Get-Service -Name "MSSQL$SQLEXPRESS"

# Если остановлен, запустите:
Start-Service -Name "MSSQL$SQLEXPRESS"
```

### 2. Примените миграции

```bash
cd WebAPI_OTK
dotnet ef database update
```

### 3. Запустите приложение

```bash
dotnet run
```

### 4. Откройте в браузере

```
http://localhost:5000
```

## Что изменилось

Теперь при открытии "Закрытие МЛ":
- ✅ Страница открывается даже если база пустая
- ✅ Показывается одно понятное сообщение вместо множества ошибок
- ✅ Можно работать с интерфейсом
- ✅ Ошибки логируются в консоль для отладки
