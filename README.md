# Система ОТК (Отдел Технического Контроля)

Web-приложение для управления маршрутными листами, операциями и премиальными коэффициентами в производственной среде.

## Технологии

### Backend
- ASP.NET Core 8.0
- Entity Framework Core
- PostgreSQL (рекомендуется) / SQL Server
- RESTful API

### Frontend
- Vanilla JavaScript (ES6+)
- HTML5
- CSS3
- Адаптивный дизайн

## Основные модули

### 1. Закрытие МЛ (Маршрутных листов)
- Поиск МЛ по номеру, изделию, ДСЕ
- Просмотр операций МЛ
- Фильтрация операций по статусу, сотруднику, датам
- Завершение операций
- Закрытие МЛ с указанием данных ОТК

### 2. Премиальные коэффициенты
- CRUD операции с коэффициентами
- Фильтрация по статусу, изделию, типу операции
- "Мягкое" удаление (деактивация активных коэффициентов)
- Физическое удаление только неактивных коэффициентов
- Статистика по коэффициентам

### 3. Каталог ДСЕ
- Просмотр деталей и сборочных единиц
- Пагинация (до 100 записей на страницу)
- Поиск по коду и наименованию
- Фильтрация по изделию

## API Endpoints

### Маршрутные листы
- `GET /api/RouteList` - Получить список МЛ
- `GET /api/RouteList/{id}` - Получить МЛ по ID
- `GET /api/RouteList/opened` - Получить открытые МЛ
- `PUT /api/RouteList/close/{id}` - Закрыть МЛ

### Операции МЛ
- `GET /api/OperationRouteList` - Получить список операций
- `GET /api/OperationRouteList/{id}` - Получить операцию по ID
- `POST /api/OperationRouteList` - Создать операцию
- `PUT /api/OperationRouteList/{id}` - Обновить операцию
- `PUT /api/OperationRouteList/finish/{id}` - Завершить операцию
- `DELETE /api/OperationRouteList/{id}` - Удалить операцию

### Премиальные коэффициенты
- `GET /api/PremKoef` - Получить список коэффициентов
- `GET /api/PremKoef/{id}` - Получить коэффициент по ID
- `POST /api/PremKoef` - Создать коэффициент
- `PUT /api/PremKoef/{id}` - Обновить коэффициент
- `PUT /api/PremKoef/deactivate/{id}` - Деактивировать коэффициент
- `DELETE /api/PremKoef/{id}` - Удалить коэффициент

### ДСЕ
- `GET /api/DCE` - Получить список ДСЕ (с пагинацией)
- `GET /api/DCE/{id}` - Получить ДСЕ по ID

### Типы операций
- `GET /api/OperationType` - Получить список типов операций
- `GET /api/OperationType/{id}` - Получить тип операции по ID

### Сотрудники
- `GET /api/Employee` - Получить список сотрудников
- `GET /api/Employee/{id}` - Получить сотрудника по ID

### Изделия
- `GET /api/Product` - Получить список изделий
- `GET /api/Product/{id}` - Получить изделие по ID
- `GET /api/Product/{id}/DCE` - Получить ДСЕ изделия
- `GET /api/Product/{id}/ML` - Получить МЛ изделия

## Запуск проекта

### Требования
- .NET 8.0 SDK
- PostgreSQL 12+ (рекомендуется 15+) или SQL Server
- Современный браузер (Chrome, Firefox, Edge)

### Быстрый старт с PostgreSQL (рекомендуется)

1. Клонировать репозиторий
```bash
git clone <repository-url>
cd WebAPI_OTK
```

2. Установить PostgreSQL
См. подробную инструкцию в `database/SETUP_POSTGRESQL.md`

3. Создать базу данных и загрузить данные
```bash
# Создать БД
psql -U postgres -c "CREATE DATABASE otk_db;"

# Выполнить схему
psql -U postgres -d otk_db -f database/schema.sql

# Загрузить тестовые данные
psql -U postgres -d otk_db -f database/data.sql
```

4. Установить Npgsql пакет
```bash
cd WebAPI_OTK
dotnet add package Npgsql.EntityFrameworkCore.PostgreSQL --version 8.0.0
```

5. Настроить строку подключения
Отредактировать `appsettings.json`:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=otk_db;Username=postgres;Password=your_password"
  }
}
```

6. Обновить Program.cs
Заменить `UseSqlServer` на `UseNpgsql`:
```csharp
builder.Services.AddDbContext<OTKContext>(options =>
    options.UseNpgsql(builder.Configuration.GetConnectionString("DefaultConnection")));
```

7. Запустить приложение
```bash
dotnet run
```

8. Открыть в браузере
```
http://localhost:5000
```

### Альтернатива: SQL Server

Если вы предпочитаете SQL Server, см. `MIGRATION_GUIDE.md` для инструкций по настройке.

## База данных

### Структура БД

Система использует следующие таблицы:
- **Должность** - справочник должностей
- **Изделие** - справочник изделий
- **ТипОперации** - справочник типов операций
- **Оборудование** - справочник оборудования
- **Сотрудник** - сотрудники предприятия
- **ДСЕ** - детали и сборочные единицы
- **МЛ** - маршрутные листы
- **Операция_МЛ** - операции маршрутных листов
- **ПремиальныеКоэффициенты** - премиальные коэффициенты
- **Зарплата** - расчет зарплаты

### SQL скрипты

В директории `database/` находятся готовые SQL скрипты для PostgreSQL:
- `schema.sql` - создание схемы БД (таблицы, индексы, представления, триггеры)
- `data.sql` - тестовые данные для демонстрации

### Тестовые данные

После выполнения `data.sql` в БД будут загружены:
- 7 должностей
- 4 изделия
- 10 типов операций
- 7 единиц оборудования
- 12 сотрудников
- 14 ДСЕ
- 8 маршрутных листов (открытые и закрытые)
- 25+ операций с разными статусами
- 9 премиальных коэффициентов (активные и неактивные)
- Примеры расчетов зарплаты

### Миграция с SQL Server

Если вы ранее использовали SQL Server и хотите перейти на PostgreSQL:
1. См. подробную инструкцию в `database/SETUP_POSTGRESQL.md`
2. Используйте готовые SQL скрипты `schema.sql` и `data.sql`
3. Обновите строку подключения в `appsettings.json`
4. Установите пакет `Npgsql.EntityFrameworkCore.PostgreSQL`
5. Замените `UseSqlServer` на `UseNpgsql` в `Program.cs`

## Особенности реализации

### Логика "мягкого" удаления коэффициентов
- Активные коэффициенты нельзя удалить физически
- При деактивации устанавливается `ДатаОкончания = текущая дата`
- Физическое удаление доступно только для неактивных коэффициентов

### Закрытие МЛ
- МЛ можно закрыть только если все операции завершены
- Дата закрытия МЛ определяется датой окончания финальной операции
- При закрытии указываются: сотрудник ОТК, количество ОТК, брак

### Пагинация ДСЕ
- Максимум 100 записей на страницу
- Поддержка поиска и фильтрации
- Оптимизация для больших объемов данных
