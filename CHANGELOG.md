# История изменений

## [1.1.0] - 2026-03-06

### Добавлено

#### Миграция на PostgreSQL
- SQL скрипт `database/schema.sql` для создания схемы БД в PostgreSQL
  - 10 таблиц с правильными типами данных PostgreSQL
  - Индексы для оптимизации запросов
  - Представления (views) для активных коэффициентов и открытых МЛ
  - Функция и триггер для автоматического расчета длительности операций
  - Комментарии к таблицам и полям
- SQL скрипт `database/data.sql` с тестовыми данными
  - 7 должностей
  - 4 изделия
  - 14 ДСЕ
  - 10 типов операций
  - 7 единиц оборудования
  - 12 сотрудников
  - 8 маршрутных листов (открытые и закрытые)
  - 25+ операций с разными статусами
  - 9 премиальных коэффициентов
  - Примеры расчетов зарплаты

#### Документация
- `database/SETUP_POSTGRESQL.md` - подробная инструкция по настройке PostgreSQL
  - Установка PostgreSQL для Windows/Linux/macOS
  - Создание БД и пользователя
  - Выполнение SQL скриптов
  - Настройка строки подключения
  - Устранение проблем
- `POSTGRESQL_QUICKSTART.md` - краткая инструкция для быстрого старта
- `.kiro/specs/postgresql-migration/requirements.md` - спецификация миграции

### Изменено

#### Конфигурация
- Обновлен `appsettings.json` с примером строки подключения PostgreSQL
  - Добавлена старая строка подключения SQL Server как альтернатива
- Обновлен `appsettings.Development.json` для разработки с PostgreSQL
  - Включено логирование SQL запросов
- Обновлен `README.md`
  - Добавлена информация о поддержке PostgreSQL
  - Обновлены инструкции по запуску
  - Добавлен раздел "База данных" с описанием структуры и тестовых данных

#### Обработка ошибок
- Улучшена обработка ошибок в `ml-closure.js`
  - Каждый справочник загружается отдельно с try-catch
  - Детальные сообщения об ошибках для каждого справочника
- Улучшена обработка ошибок в `prem-koef.js`
  - Аналогичная логика раздельной загрузки
- Изменен `api.js`
  - Убраны автоматические toast-уведомления для всех ошибок
  - Toast показывается только для критичных ошибок (сеть, сервер)

#### Документация по устранению ошибок
- Обновлен `FIX_500_ERROR.md` с информацией о PostgreSQL
- Обновлен `DEBUGGING.md` с новыми сценариями
- Создан `QUICK_FIX.md` для быстрого решения проблем

### Технические детали

#### Преимущества PostgreSQL
- Кроссплатформенность (Windows, Linux, macOS)
- Бесплатная open-source СУБД
- Лучшая производительность для сложных запросов
- Более строгая типизация данных
- Поддержка JSON, массивов, полнотекстового поиска
- Активное сообщество и документация

#### Изменения в типах данных
- `SERIAL` вместо `IDENTITY` для автоинкрементных ID
- `VARCHAR` вместо `NVARCHAR`
- `TIMESTAMP` вместо `DATETIME2`
- `DATE` для дат без времени
- `DECIMAL` для денежных значений

#### Требования для миграции
- PostgreSQL 12+ (рекомендуется 15+)
- Npgsql.EntityFrameworkCore.PostgreSQL 8.0+
- Обновление `Program.cs`: замена `UseSqlServer` на `UseNpgsql`

### Исправлено
- Устранены ошибки HTTP 500 при отсутствии БД
  - Теперь показываются понятные сообщения об ошибках
  - Каждый справочник загружается независимо
- Улучшена обработка ошибок сети
- Добавлены подробные инструкции по устранению проблем

## [1.0.0] - 2026-03-06

### Добавлено

#### Backend API
- Контроллер `PremKoefController` с полным CRUD для премиальных коэффициентов
  - Логика "мягкого" удаления (деактивация через установку ДатаОкончания)
  - Физическое удаление только неактивных коэффициентов
- Контроллер `DCEController` с пагинацией (макс 100 записей)
- Контроллер `OperationTypeController` для получения типов операций
- Эндпоинт `PUT /api/RouteList/close/{id}` для закрытия МЛ
- DTO для всех новых контроллеров

#### Frontend - Инфраструктура
- Настройка ASP.NET для раздачи статических файлов
- API клиент (`api.js`) с методами для всех модулей
- Утилиты (`utils.js`): форматирование, валидация, дебаунс
- Переиспользуемые компоненты (`components.js`): toast, loader, modal, таблицы
- Цветовая схема: #FFFD82, #0A0903, #EFCEFA, #C89B7B, #2F3E46
- Главная страница с навигацией и карточками модулей
- Заглушка авторизации (принимает любые непустые логин/пароль)

#### Frontend - Модуль закрытия МЛ
- HTML страница `ml-closure.html`
- Форма поиска МЛ (по номеру, изделию, ДСЕ)
- Отображение списка МЛ и детальной информации
- Фильтры операций (статус, сотрудник, даты)
- Таблица операций с кнопкой завершения
- Модальное окно закрытия МЛ (сотрудник ОТК, количество, брак)
- JavaScript логика `ml-closure.js`

#### Frontend - Модуль премиальных коэффициентов
- HTML страница `prem-koef.html`
- Панель статистики (всего/активных/неактивных)
- Фильтры: поиск, статус, изделие, тип операции, дата проверки
- Модальное окно создания/редактирования
- JavaScript логика `prem-koef.js` с CRUD операциями
- Отображение коэффициентов в виде карточек

#### Frontend - Модуль каталога ДСЕ
- HTML страница `dce-catalog.html`
- Таблица с пагинацией
- Поиск по коду и наименованию
- Фильтр по изделию
- Модальное окно с деталями ДСЕ
- JavaScript логика `dce-catalog.js`

#### Стили
- `main.css` - основные стили, переменные, компоненты
- `components.css` - модальные окна, toast, loader, таблицы
- `pages.css` - стили для страниц, карточки, пагинация
- Адаптивный дизайн для мобильных устройств
- Улучшения доступности (фокус, скринридеры, высокая контрастность)

#### Документация
- `README.md` - полное описание проекта, API, запуск
- `MIGRATION_GUIDE.md` - руководство по миграциям БД
- `CHANGELOG.md` - история изменений

- Создана миграция `RemoveDateClosedFromML`

#### API
- Улучшена обработка ошибок в API клиенте
- Добавлена проверка ошибок сети
- Улучшены сообщения об ошибках

### Исправлено
- Убрана неиспользуемая переменная `modal` в `prem-koef.js`
- Добавлена загрузка типов операций в модуле премиальных коэффициентов

## Технические детали

### Архитектура
- Backend: ASP.NET Core 8.0 + Entity Framework Core + SQL Server
- Frontend: Vanilla JavaScript + HTML5 + CSS3
- API: RESTful
- Аутентификация: Заглушка (для демо)

### Основные функции
1. Закрытие маршрутных листов с проверкой завершения операций
2. CRUD премиальных коэффициентов с "мягким" удалением
3. Просмотр каталога ДСЕ с пагинацией

### Особенности реализации
- Логика "мягкого" удаления коэффициентов
- Пагинация для больших объемов данных (ДСЕ)
- Адаптивный дизайн для всех устройств
- Улучшенная доступность (WCAG)
- Обработка ошибок сети и API
