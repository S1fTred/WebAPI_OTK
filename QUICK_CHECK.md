# Быстрая проверка базы данных

## Что было сделано

Добавлены три новых поля дат для операций:
- **ДатаВыдачи** - плановая дата начала (левая граница)
- **ДатаИсполнения** - фактическая дата завершения
- **ДатаЗакрытия** - дедлайн (правая граница)

## Как проверить

### 1. Запустите сервер из Rider IDE

В консоли должны появиться сообщения:
```
Добавлено 26 операций к МЛ
Обновлено X операций с новыми датами
```

### 2. Проверьте файл базы данных

Запустите в PowerShell:
```powershell
./check_database_simple.ps1
```

Должен появиться файл `WebAPI_OTK/otk.db`

### 3. Проверьте API

Откройте в браузере (при запущенном сервере):
```
http://localhost:5000/api/OperationRouteList?млId=1
```

В JSON должны быть поля:
```json
{
  "датаВыдачи": "2026-02-14T08:00:00",
  "датаИсполнения": "2026-02-15T11:30:00",
  "датаЗакрытия": "2026-02-18T08:00:00"
}
```

### 4. Проверьте интерфейс

1. Откройте: `http://localhost:5000/ml-closure.html`
2. Войдите в систему
3. Откройте любой МЛ
4. В таблице операций должны быть три новые колонки:
   - Дата Выдачи
   - Дата Исполнения
   - Дата Закрытия

## Если что-то не работает

Удалите базу данных и перезапустите сервер:
```powershell
Remove-Item WebAPI_OTK/otk.db*
```

Затем запустите сервер снова из Rider IDE.
